## What this is

This project studies how gravitational systems transition from predictable to chaotic behaviour. It starts with classical two-body mechanics — where analytical solutions exist — and extends into the three-body regime, where no closed-form solution exists and long-term prediction becomes fundamentally unstable.

Three components are tightly coupled throughout:

| Component | Role |
|-----------|------|
| **Physics** | Newtonian gravity, conserved quantities, orbital mechanics |
| **Numerics** | Symplectic integration (Velocity Verlet) for energy-stable long-run simulation |
| **Machine Learning** | Physics-informed feature extraction, stability classification, eccentricity regression |

## Physics Background

### Two-body (Phase 1–2)
The two-body problem is analytically solvable via Kepler's laws. It is used here as a validation baseline — the integrator is correct when energy and angular momentum are conserved to machine precision over thousands of orbits.

### Three-body (Phase 3)
No general analytical solution exists. Key properties:

- Small perturbations in initial conditions grow **exponentially** over time
- Systems exhibit **sensitive dependence on initial conditions** (classical chaos)
- Long-term trajectories are bounded but unpredictable

Chaos is quantified using **MEGNO** (Mean Exponential Growth of Nearby Orbits):

```
⟨Y⟩ ≈ 2   →   quasi-periodic / stable
⟨Y⟩ > 2   →   chaotic
⟨Y⟩ >> 2  →   strongly unstable (ejection or collision likely)
```

## Dataset

Datasets are generated by sampling across three physically distinct regimes:

| Regime | Typical behaviour | Sampling rationale |
|--------|------------------|--------------------|
| **Hierarchical** | Mostly stable — one tight pair + distant third body | Lower-chaos baseline |
| **Asymmetric** | Mixed — moderate mass ratio asymmetry | Transition region |
| **Near-equal** | Highly chaotic — comparable masses, strong coupling | Upper-chaos boundary |

Each sample stores:

- Initial conditions (positions, velocities, masses)
- Full trajectory array
- Energy and angular momentum time series
- MEGNO value
- Outcome label (`stable` / `chaotic` / `unstable`)

> Datasets are not included in this repository due to size. Run `dataset_generator3.py` to regenerate them locally.

## Machine Learning

### Experiment A — Eccentricity Regression (2-body)

Predict orbital eccentricity from physics-based features extracted at simulation time.

Features include specific orbital energy `ε`, specific angular momentum `h`, and their normalised combinations. These are physically meaningful quantities rather than raw coordinate inputs.

### Experiment B — Stability Classification (3-body)

Classify each simulated system into one of three outcomes:

- `stable` — regular, quasi-periodic motion
- `chaotic` — bounded but unpredictable long-term behaviour
- `unstable` — body ejection or collision within the simulation window

**Key finding:** Physics-informed features consistently outperform raw coordinate inputs and generalise better across regimes. A model trained on hierarchical systems transfers more reliably to near-equal systems when features are physics-normalised.

## Visualisation

The `visualize3.py` module produces an animated diagnostic dashboard for any simulation result:

| Panel | What it shows |
|-------|--------------|
| **Orbital trajectories** | Fading-trail animation of all three bodies |
| **Conservation score** | `-log₁₀(max(∣ΔE∣, ∣ΔL∣))` — integrator precision in digits |
| **Pair distances** | r₁₂, r₁₃, r₂₃ over time — reveals close approaches |
| **Phase portrait** | x₁ vs ẋ₁ — closed loop = regular, tangled = chaotic |
| **Angular momentum drift** | Cumulative ΔL/\|L₀\| — measures long-run numerical leakage |
| **CoM drift** | \|com(t) − com(0)\| — should stay ≈ 0 for an isolated system |
